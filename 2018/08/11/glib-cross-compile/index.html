glib-2.46
https://github.com/GNOME/glib/tree/glib-2-46
unzip glib-glib-2-46.zip
cd glib-glib-2-46/
gedit glib.cache:
glib_cv_long_long_format=ll
glib_cv_stack_grows=no
glib_cv_have_strlcpy=no
glib_cv_have_qsort_r=yes
glib_cv_va_val_copy=yes
glib_cv_uscore=no
glib_cv_rtldglobal_broken=no
ac_cv_func_posix_getpwuid_r=yes
ac_cv_func_posix_getgrgid_r=yes

apt-get install autoconf
apt-get install libtool
apt-get install pkg-config
apt-get install zlib1g-dev
apt-get install libffi-dev
apt-get install gettext

./autogen.sh
configure.ac:88: error: possibly undefined macro: AC_DEFINE
get this done on a virtual machine with exact the same path and same debian version like debian server.then move the entire updated source code to the server.packed as glib.tar

step1: run a x86 build which is needed by cross compilation later.
tar xvf glib.tar
cd glib-glib-2-46
mkdir install
mkdir install/pc
./configure –prefix=/home/xxx/NDC/06.00.00.00/arm-none-linux-gnueabi-4.9.1/mpc/glib/glib-glib-2-46/install/pc –cache-file=glib.cache
make
make install

step2: run an arm build from another clean build.
cd ..
mv glib-glib-2-46 glib-glib-2-46-pc
tar xvf glib.tar
cd glib-glib-2-46
export PATH=$PATH:/home/xxx/NDC/06.00.00.00/arm-none-linux-gnueabi-4.9.1/mpc/glib/glib-glib-2-46-pc/install/pc/bin

to avoid the error by several tests inside configure can not run for cross compiling situation:
edit configure script
need to make the test code to return 0 . and change the
if test “$cross_compiling” = yes
to
if test “$cross_compiling” = no
in order to cheat configure script as the test has passed.
save as configure_arm
very nasty shitty work here.

to avoid the error ./.libs/libgobject-2.0.so: undefined reference to `ffi_type_pointer’ when running make:
cp /home/xxx/NDC/06.00.00.00/arm-none-linux-gnueabi-4.9.1/mpc/glib/build/lib/libffi.so /home/mgc/embedded/codebench/arm-none-linux-gnueabi/libc/usr/lib/
cp /home/share/zlib/1.2.11/Board/arm-none-linux-gnueabi-4.9.1/lib/libz.so /home/mgc/embedded/codebench/arm-none-linux-gnueabi/libc/usr/lib/
remember to delete those libs after make is done

./configure –prefix=/home/xxx/NDC/06.00.00.00/arm-none-linux-gnueabi-4.9.1/mpc/glib/glib-glib-2-46/install/arm CC=arm-linux-gcc –host=arm-none-linux-gnueabi LIBFFI_CFLAGS=”-I/home/xxx/NDC/06.00.00.00/arm-none-linux-gnueabi-4.9.1/mpc/glib/build/lib/libffi-3.2.1/include” LIBFFI_LIBS=”-lffi -L/home/xxx/NDC/06.00.00.00/arm-none-linux-gnueabi-4.9.1/mpc/glib/build/lib” ZLIB_CFLAGS=”-I/home/share/zlib/1.2.11/Board/arm-none-linux-gnueabi-4.9.1/include” ZLIB_LIBS=”-lz -L/home/share/zlib/1.2.11/Board/arm-none-linux-gnueabi-4.9.1/lib” –cache-file=glib.cache –disable-selinux –disable-xattr –disable-libelf

make
make install

result is in /home/xxx/NDC/06.00.00.00/arm-none-linux-gnueabi-4.9.1/mpc/glib/glib-glib-2-46/install/arm
