telnet deploy

文件系统加入telnet服务

http://blog.chinaunix.net/uid-9255716-id-107976.html busybox里面已经集成了，先参考这篇文章。
步骤
tar xvfj busybox-1.20.0_for_TQ3358_V1.0.tar.bz2 -C /
cp -f config_TQ335x_V1.0 .config
make menuconfig 在Networking Utilities里面找到telnet和telnetd。发现已经勾上了
make
make install
所以再到rootfs里面看，发现有telnetd了。
然后修改开发板的/etc/init.d/rcS

#!/bin/sh
PATH=/sbin:/bin:/usr/sbin:/usr/bin
runlevel=S
prevlevel=N
umask 022
export PATH runlevel prevlevel
mount -a
mkdir /dev/pts            注意加入这行
/bin/mknod /dev/pts/0 c 136 0    注意加入这行
/bin/mknod /dev/pts/1 c 136 1    注意加入这行
/bin/mknod /dev/pts/2 c 136 2    注意加入这行
/bin/mknod /dev/pts/3 c 136 3    注意加入这行
/bin/mknod /dev/pts/4 c 136 4    注意加入这行
/bin/mknod /dev/pts/5 c 136 5    注意加入这行
/bin/mount -t devpts devpts /dev/pts    注意加入这行
echo /sbin/mdev>/proc/sys/kernel/hotplug
mdev -s
/bin/hostname -F /etc/sysconfig/HOSTNAME
/sbin/ifconfig lo 127.0.0.1
/sbin/ifconfig eth0 192.168.1.168
/sbin/route add default gw 192.168.1.1
/etc/init.d/ts.sh
/usr/sbin/telnetd            注意加入这行

http://blog.csdn.net/xautfengzi/article/details/6250133
http://blog.sina.com.cn/s/blog_40aad2950100h1zl.html

这样重启，板子就有telnet服务了。此时在虚拟机里面输入：
telnet 192.168.1.6
显示：
Trying 192.168.1.6…
Connected to 192.168.1.6.
Escape character is ‘^]’.

EmbedSky login:
输入root，回车，就登录进去了。
然后要退出的话，按ctrl+D即可。

telnet登录

发现没有/usr/sbin/telnetd
修改：
~/NUC977/nuc970bsp-release-20151111/BSP/busybox-1.22.1
make install
客户端有这个错：
Trying 159.99.164.44…
Connected to 159.99.164.44.
Escape character is ‘^]’.
Connection closed by foreign host.

参考http://blog.csdn.net/zxh821112/article/details/8593650

telnetd -F跑前台，看看出什么错：
telnetd: can’t find free pty

busybox选项：
CONFIG_FEATURE_DEVPTS=y
CONFIG_TELNETD=y
FEATURE_SUID
内核选项：
CONFIG_UNIX98_PTYS=y
CONFIG_DEVPTS_MULTIPLE_INSTANCES=y

没用，虽然telnetd: can’t find free pty这个错消失了，但是还是Connection closed by foreign host.
偶然发现，busybox的menuconfig的telnetd选项的Help里面，有大段的下面的帮助文档：

Note that for busybox telnetd to work you need several things: \
\ First of all, your kernel needs: \
\ UNIX98_PTYS=y \
\ DEVPTS_FS=y \
\ \
\ Next, you need a /dev/pts directory on your root filesystem: \
\ \
\ $ ls -ld /dev/pts \
\ drwxr-xr-x 2 root root 0 Sep 23 13:21 /dev/pts/ \
\ \
\ Next you need the pseudo terminal master multiplexer /dev/ptmx: \
\ \
\ $ ls -la /dev/ptmx \
\ crw-rw-rw- 1 root tty 5, 2 Sep 23 13:55 /dev/ptmx \
\ \
\ Any /dev/ttyp[0-9]* files you may have can be removed. \
\ Next, you need to mount the devpts filesystem on /dev/pts using: \
\ \
\ mount -t devpts devpts /dev/pts \
\ \
\ You need to be sure that busybox has LOGIN and \
\ FEATURE_SUID enabled. And finally, you should make \
\ certain that Busybox has been installed setuid root: \
\ \
\ chown root.root /bin/busybox \
\ chmod 4755 /bin/busybox \
\ \
\ with all that done, telnetd should work…. \
\

接着改：
busybox选项：
FEATURE_SUID=y
LOGIN=y
这次可以了，但无法login成功。
telnet 159.99.164.44
Trying 159.99.164.44…
Connected to 159.99.164.44.
Escape character is ‘^]’.

(none) login: root
Password:
Login incorrect
busybox选项：
WHOAMI=y
PASSWD=y
然后用passwd设置了一个root密码后，可以用telnet登录root了。
能否像iq8那样，不用密码登录root？
用passwd设置空密码，telnet才可以root无密码登录的
这种情况下，/etc/passwd会发生改变
把修改过密码为空的这个文件，拷入编译服务器的rootfs里面。
这样就可以不用密码telnet登录root了。
