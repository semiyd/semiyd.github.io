u-boot device tree

有关u-boot的设备树，分两块来讲。
一是最常见的，u-boot支持fdt的加载，把dtb传递给内核。
二是比较新出现的，u-boot本身也自带一个小dts的情况。

u-boot传递kernel的dtb给kernel

编译uboot前加入 #define CONFIG_OF_LIBFDT /* Device Tree support */
根据mkconfig：
make TQ3358_config
解析出BOARD_NAME=”${1%_config}”也就是BOARD_NAME=TQ3358

所以需要改TQ3358.h 加入#define CONFIG_OF_LIBFDT
export PATH=/usr/local/arm/gcc-linaro-arm-linux-gnueabi-2012.03-20120326_linux/bin
编译
这样就支持device tree了，包括fdt命令。
当前的uboot尚不支持bootz
所以kernel编译需要修改Makefile为：
$(MAKE) -j $(MAKE_JOBS) -C $(LINUXKERNEL_INSTALL_DIR) ARCH=arm CROSS_COMPILE=$(CROSS_COMPILE) uImage LOADADDR=0x80008000
其中0x82000000就是uboot的cpaddr环境变量值。
环境变量加入：
tftp 600000 mpc8315erdb.dtb
cp.b 600000 fff80000 $filesize
也许需要（设置fdt的地址）：fdt addr 0x71000000，这个地址需要和tftp 下载到的RAM地址一致。
fdtfile=mpc8315erdb.dtb

tftp 1000000 rootfs.ext2.gz.uboot // 下载uboot可用的文件系统，在ltib目录下
tftp 200000 uImage // 下载内核镜像
tftp 800000 mpc8313erdb.dtb // 下载dtb文件
bootm 200000 1000000 800000 // 启动

load mmc 0 0x88000000 /boot/am335x-evm.dtb
load mmc 0 0x82000000 /boot/zImage
bootz 0x82000000 - 0x88000000
对于 ARM，使用 bootz ${loadaddr} ${rdaddr}:${rdsize} ${fdtaddr} 命令来启动 Kernel。

加载dtb
tftp 0x88000000 am335x-evm.dtb
fdt addr 0x88000000
尝试启动kernel
tftp 0x80007fc0 uImage
bootm 0x80007fc0 - 0x88000000

cpaddr和load address的关系
load address=Entry Point？

原来3.2.0的情况，是这样的：
cpaddr=0x82000000 tftp的位置
nand addr kernel = 0x580000 nand里面存kernel的位置
kloadaddr=0x80007fc0 nand读出来kernel到ddr的位置
LOADADDR=0x80008000 uImage跳过64字节后真正开始执行的位置
这种中间和kloadaddr=0x80007fc0差了64
uImage就是zImage加上一个64的头部。头部要跳过。
所以kloadaddr必须是0x80007fc0

改动uboot下载和启动部分
1.增加sd卡读取dts并保存
替换原先menu里面的6号，也就是烧写yaffs，改成烧写dtb
dtb的实际大小：36205字节，也就是36K。并且看了arch/arm/boot/dts下面的dtb文件，没有超过100KB的。
“mmc rescan;fatload mmc 0 ${cpaddr} ${rootimgname};nandecc hw 0;nand erase root;nand write.yaffs2 ${cpaddr} root $(filesize)”
cpaddr属于环境变量，cpaddr=0x82000000，看代码，不管是烧uImage还是root.bin，都是先拷入cpaddr的，相当于一个临时存放的地址，所以这个值不用改。
rootimgname改为dtbimgname，并加入CONFIG_EXTRA_ENV_SETTINGS这个宏里面
nandecc参考uimage，用的也是hw 0。所以我们也用hw 0。
nand erase root这句话，root是个地址定义？
根据nand –help的结果：
nand write - addr off|partition size
也就是说，nand write后面跟的参数是： 1.从哪里拷贝(RAM地址) 2.偏移/分区名字 3.拷贝大小。
最好还是使用分区来表示，这样比较直观。
MTDPARTS_DEFAULT目前的定义是”mtdparts=TQ3358-nand:128k@0(SPL),128k(SPL.backup0),128k(SPL.backup1),128k(SPL.backup2),1920k(uboot),128k(params),3m(logo),5m(kernel),-(root)”
仔细看了下，这个定义和kernel里面的分区定义确实是一致的。
所以目前看，其实把root分区，替换成SPL.backup2应该是可以的，换个名字，这样也不用动分区结构了。
最终改成：
“mmc rescan;fatload mmc 0 ${cpaddr} ${dtbimgname};nandecc hw 0;nand erase DTB;nand write.i ${cpaddr} DTB $(filesize)”
2.改动bootm
a.先读取dts，加入fdt addr ，再读取uimage
b.bootm命令参数改变

把uimage加载到0x80007fc0地址
把nand的dtb加载到0x88000000地址
bootm 0x80007fc0 - 0x88000000

是这个nand_boot，在CONFIG_EXTRA_ENV_SETTINGS里面的。
“nand_boot=echo Booting from nand …; “ \
“nandecc hw 0; “ \
“nand read.i ${kloadaddr} kernel ; “ \
“bootm ${kloadaddr}\0” \
改成：
“nand_boot=echo Booting from nand …; “ \
“nandecc hw 0; “ \
“nand read.i ${kloadaddr} kernel ; “ \

“fdt addr ${dtbloadaddr} ; “ \
“bootm ${kloadaddr} - ${dtbloadaddr}\0” \

make TQ3358_config
make -j8

可以启动，没问题

u-boot使用自己的dtb用于启动

比较新的u-boot，例如u-boot-2017，都是使用的kbuild的编译架构，同时也使用了设备树。
u-boot所使用的设备树，是给自己启动用的，和内核所用的设备树，其实是没有关联的。但在设备树本身的格式上相近，都属于dts的格式。
u-boot因为在驱动上很多方面和内核驱动非常接近，所以借用了内核的设备树的思路，但是在实际代码上和内核是独立的。
以am57xx平台为例，u-boot的dtb编译在u-boot镜像里面，而kernel的dtb则放在rootfs的/boot下面。

设备树位于：
arch/arm/dts/

在CONFIG_SPL_LOAD_FIT的配置下，首先u-boot.bin会和dtc编译出来的dtb文件，一起生成u-boot.img这个FIT镜像。
而SPL阶段，会去解析这个FIT镜像，根据检测出来的所在的板子型号，来选择其中正确的dtb，并把这个dtb在引导u-boot启动的时候，传递给u-boot
dts和configs/xxx_board_defconfig合起来，可以决定如何对特定的板卡做初始化。
u-boot的dts内容较为简单，大多是初始化一些电压，网络，内存大小，内存接口，usb接口等仅仅在u-boot阶段需要初始化的东西。
可以参考”am571x-idk.dts”和”am57xx-idk-common.dtsi”

u-boot.img的生成过程：
tools/mkimage -f auto -A arm -T firmware -C none -O u-boot -a 0x80800000 -e 0 -n “U-Boot 2017.01 for am57xx board” -E -b arch/arm/dts/am57xx-beagle-x15.dtb
-b arch/arm/dts/am57xx-beagle-x15-revb1.dtb -b arch/arm/dts/am57xx-beagle-x15-revc.dtb -b arch/arm/dts/am57xx-evm-reva3.dtb -b arch/arm/dts/am572x-idk.dtb
-b arch/arm/dts/am571x-idk.dtb -b arch/arm/dts/am574x-idk.dtb -d u-boot-nodtb.bin u-boot.img >/dev/null
可见：u-boot-nodtb.bin + dtb文件 = u-boot.img

还有一种编译方法：
make DEVICE_TREE=zynq-zed -j4
