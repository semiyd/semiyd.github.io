IMX6ULL DDR3调试

Posted on 2018-08-12 | In Linux , U-boot , 驱动 , DDR

IMX6ULL DDR3调试

ddr怎么调试？
参考文档：
Freescale i.MX6 DRAM Port Application Guide-DDR3.pdf
K4B2G1646C-HCH9.pdf
名词解释：
Multi Mode DDR Controller (MMDC)
drive strength (DSE)
DDR内部Organization方式：
DDR大小一般用bit来表示。比如2Gb，其实是256MB。
那么这个总大小，由三部分组成。
以MT41K128M16为例：
16Mb x 16 x 8banks = 2Gb
也就是说，orgranization是16Mb x 16，而bank数是8，所以总数是2Gb
那么同样是2Gb的总大小，其实可以有各种不同的organization，例如：
32Mb x 8 x 8banks
64Mb x 4 x 8banks
又以K4B2G1646C-HCH9为例，这片一样是2Gb总大小。
打开spec，搜HCH9，能看到spec写明，这芯片是DDR-1333，并且是128Mx16。这个写法和美光的略有不同。128Mx16=2Gb
在spec里面，写明了The 2Gb DDR3 SDRAM C-die is organized as a 16Mbit x 16 I/Os x 8 banks。
其实128Mx16就是=16Mb x 16 x 8banks.
使用I.MX6ULL_DDR3_Script_Aid_V0.01.xlsx生成初始参数realview.inc
准备电路图，ddr spec
Register Configuration页，所有橙色和蓝色的框都是要填的
Device information类：
根据Freescale i.MX6 DRAM Port Application Guide-DDR3.pdf的具体步骤，从spec里面找相应的参数，填入excel表。
K4B2G1646C-HCH9 ：
填上Manufacturer和Memory part number
Memory type，根据spec,HCH9后缀对应的是DDR3-1333(9-9-9)，选上1333.
DRAM density (Gb)，芯片是2Gb的，选上2
DRAM Bus Width，一般都是16bit，选16
Number of Banks，根据spec page9，2Gb那个表格的128Mb x 16这一列，选8
Number of ROW Addresses，根据spec page9，2Gb的128Mb x 16这一列,对应是A0- A13，也就是选14
Number of COLUMN Addresses，根据spec page9，2Gb的128Mb x 16这一列,对应是A0- A9，也就是选10
Page Size (K)，根据spec page9，2Gb的128Mb x 16这一列,对应是2KB，也就是选2
tRCD=tRP=CL (ns)，根据spec page5，[ Table 2 ] 2Gb DDR3 C-die Speed bins的DDR3-1333这一列，填13.5
tRC Min (ns)，根据spec page5，[ Table 2 ] 2Gb DDR3 C-die Speed bins的DDR3-1333这一列，填49.5
tRAS Min (ns)，根据spec page5，[ Table 2 ] 2Gb DDR3 C-die Speed bins的DDR3-1333这一列，填36
System information类：
根据电路图来填
K4B2G1646C-HCH9 ：
Density per chip select (Gb)，应该是指每个cs对应的ddr容量，所以选2
Number of Chip Selects used选1，因为仅用了CS0。
Address Mirror (for CS1)，默认选disable，感觉不要改了，反正也没用cs1。
SI consideration类也就是信号完整性方面的：
根据PCB vendor的结果提供，包括驱动强度，ODT。具体怎么设置，要问fae。
前四个都是DSE设置，最后一个是ODT的设置。
DRAM DSE Setting - DQ/DQM (ohm)
DRAM DSE Setting - ADDR/CMD/CTL (ohm)
DRAM DSE Setting - CK (ohm)
DRAM DSE Setting - DQS (ohm)
System ODT Setting (ohm)
这部分经确认，完全有硬件这边，pcb厂家提供具体的阻抗值。

delay line的值
包括Write Leveling delay, Read DQS Gating Delay, Read Delay, Write Delay，对应excel表格RealView.inc的红色格子部分。
最终在DDR_Tester.exe做完校正之后，这些值会更新到最优值。
初始值是固定的。

然后打开C:\Document\Document\IMX6ULL\NXP\DDR调试\ddr_stress_tester_v2.70.exe\ddr_stress_tester_v2.70\ddr_stress_tester_v2.70\DDR_Tester.exe（有两个功能，calibration和stress test）
选择sd模式，先连接usb线，然后上电。windows会提示找到usb输入设备。如果需要重新开始校准，拔掉电源，并拔掉usb线，再来一遍。
Load Init Script选择刚刚生成的realview.inc，怎么选择？
复制出来，存成txt格式。存储路径不能包含中文。
点download。
TARGET选择MX6ULL
DDR Density注意单位是Byte，所以例如可以选256MB.
在ddr calibration区域：
ddr freq设置成400
在ddr stress test区域：
start 设置成400
end设置成400*1.15=460
MR1 Value填写excel表格里面生成的值的高2字节。一般是0004.
让板子从usb otg模式启动，然后点Download
然后点calibration。过程长达十分钟左右。
在calibration完成以后，参考I.MX6ULL_DDR3_Script_Aid_V0.01.xlsx的realview.inc页。这个页面里面的寄存器值，以下部分，会因为calibration而发生变化。
setmem /32 0x021b080c = 0x00000004

//Read DQS Gating calibration
setmem /32 0x021b083c = 0x00000000 // MPDGCTRL0 PHY0

//Read calibration
setmem /32 0x021b0848 = 0x40404040 // MPRDDLCTL PHY0

//Write calibration
setmem /32 0x021b0850 = 0x40404040 // MPWRDLCTL PHY0
其余的值保持不变。
把上面这些值。写入Realview.inc
然后从Realview.inc，写入uboot\board\freescale\mx6ul_14x14_evk\imximage.cfg(Realview.inc其实和imximage.cfg的整个寄存器的顺序都是一模一样的，直接抄过去。)

校准结束后，从校准结果中，翻到最后，可以找到：
MMDC registers updated from calibration

Write leveling calibration
MMDC_MPWLDECTRL0 ch0 (0x021b080c) = 0x00000006
MMDC_MPWLDECTRL1 ch0 (0x021b0810) = 0x000F000F

Read DQS Gating calibration
MPDGCTRL0 PHY0 (0x021b083c) = 0x01640158
MPDGCTRL1 PHY0 (0x021b0840) = 0x00000000

Read calibration
MPRDDLCTL PHY0 (0x021b0848) = 0x40403032

Write calibration
MPWRDLCTL PHY0 (0x021b0850) = 0x40403A30
这些就是校准过的delay line的值了。
现在结合这些delay line值，并且完全照搬RealView.inc里面的内容，全部写到imximage.cfg里面去。
修改/home/xxx/IOT/grip-hybird/uboot/board\xxx\adt-aio_200-01875a\imximage.cfg
用Ctrl+H的办法，把setmem /32全部替换成DATA 4。并且把 = 全部替换成一个空格
但还是要检查下，会有Ctrl+H替换不到的。或者应该是一个空格，其实是两个空格的情形。
wait = on编译不过，需要注掉。

另外，如果calibration失败，那么针对那个失败的字节（指result[xx]里面的某个字节一直不变成0），需要微调那个byte的DQS和DQ对应的驱动强度。
驱动强度需和布线阻抗对应。

然后，点Stress Test，跑个24小时做压力测试。频率范围Start和End 就设置为实际要用的频率即可。

同时也要用memtest进行进一步的测试，在系统可以启动后。

注：
realview.inc里面可以看到好多，会多次往同一个寄存器写不同的值。其实作用是用作给DDR发命令，往一些寄存器例如MDSCR里面写不同的值，就是发不同的命令。

生产之前，取5块板子，每个寄存器的每个byte做平均，取个均值。
