电阻屏+片内TP控制器：

初始化module_init部分：
分配一个input_dev结构体
对分配到的结构体设置set_bit()可以产生EV_KEY事件，EV_ABS事件和BTN_TOUCH事件
设置ADC输出给input子系统的参数范围input_set_abs_params()
向input子系统注册刚刚的结构体input_register_device()
使能cpu内部供给TP控制器的时钟源
映射TP控制寄存器的地址ioremap()，对寄存器做相应的初始化
申请触屏中断和ADC中断
初始化判断滑动用的timer

ADC中断处理部分：
进入这个中断表示ADC转换已经完成
检查下如果此时触摸笔已松开，则丢弃这次结果，可以滤除小的误触。
input_report_abs上报事件，压力值为0
input_report_key
input_sync
否则经过多次重复再进这个ADC中断，得到多次的值，取平均以后，最后才上报事件
input_report_abs(s3c_ts_dev, ABS_X, x);
input_report_abs(s3c_ts_dev, ABS_Y, y);
input_report_abs(s3c_ts_dev, ABS_PRESSURE, 1);
input_report_key(s3c_ts_dev, BTN_TOUCH, 1);
input_sync(s3c_ts_dev);//表示事件结束，此时input子系统会把刚刚的3个时间打包到一个evdev包里面，通过/dev/input/eventX输出出去
启动定时器处理长按/滑动的情况

TP中断处理部分：
进入这个中断表示有按下笔或者抬笔的动作
如果是pen up，则重新进入等待中断的状态。
如果是pen down，则开启ADC开始计算坐标。

判断滑动的定时器部分：
检查下如果此时触摸笔已松开，则丢弃这次结果，可以滤除小的误触。
input_report_abs上报事件，压力值为0
input_report_key
input_sync
否则再次启动ADC

退出module_exit部分：
释放中断
解除地址映射iounmap
input_unregister_device注销input_dev
input_free_device释放input_dev
删除timer

判断是连续滑动还是长按是上层来判断的。

电阻屏+片外I2C控制器：

这种电阻屏+片外I2C控制器的配置的驱动部分，实质上就是一个i2c从设备的驱动。

i2c_client:
声明i2c_board_info{}
MACHINE_START的.init_machine函数注册i2c_register_board_info()

i2c_driver:
初始化部分：
module_init
定义i2c_driver{}
定义.name，.probe，.remove，.suspend，.resume，.id_table
其中，.suspend和.resume涉及了睡眠和唤醒的内容
i2c_add_driver()
自动匹配id后，调用i2c_driver{}的probe函数。在probe里面：
用i2c_smbus_read_byte_data()来探测触摸屏i2c从设备芯片id
input_allocate_device()分配一个input_dev{}
input_set_abs_params()设置具体的参数
INIT_DELAYED_WORK初始化后面要用到的work queue
input_register_device()注册input_dev{}
request_irq()申请触摸屏的中断
device_init_wakeup()设置为can wakeup，设置唤醒
中断处理部分：
关中断disable_irq_nosync
schedule_delayed_work()延后比如HZ/100的时间去使用work queue
work queue处理部分：
通过container_of()取得私有的结构体，里面包含i2c_client，input_dev和irq。这些后面都要用到。
依靠i2c_client{}，用i2c读取相关坐标数据
依靠input_dev{}，用input_report_abs(), input_report_key(),input_sync()上报坐标
使能irq号的中断
有关私有的部分的处理：
定义一个私有的结构体，里面包括了i2c_client，input_dev，work queue和irq
然后在probe的时候:
通过dev_set_drvdata把这个私有的结构体放到i2c_client{}->dev->driver_data里面。
在request_irq时，最后一个入参void dev_id，被赋予了这个私有的结构体。
input_set_drvdata把私有的结构体放入input_dev{}里面。（虽然实际上后面没有通过这个途径取出）
后续会使用的到这个私有结构体的地方有：
work queue里面用container_of取出，用到私有结构体里面的i2c_client{}，input_dev{}和irq中断号
i2c_driver{}的remove函数，用dev_get_drvdata取出，注销的时候需用到input_dev{}和irq中断号
i2c_driver{}的suspend/resume函数，用dev_get_drvdata取出，需用到irq中断号
中断处理函数的第二个入参，void dev_id，就是这个私有结构体的指针，所以在中断里去开启work queue排队干活的时候，会用到work queue的结构体

电容屏+片外I2C控制器：

电容屏和电阻屏的区别在于电容屏支持多点触摸

基本和上面的电阻屏+片外I2C控制器的驱动类似，与之有区别的地方有：
i2c_driver{}的.probe里面，
set_bit(ABS_MT_TOUCH_MAJOR, input_dev->absbit);
set_bit(ABS_MT_POSITION_X, input_dev->absbit);
set_bit(ABS_MT_POSITION_Y, input_dev->absbit);
set_bit(ABS_MT_WIDTH_MAJOR, input_dev->absbit);
set_bit(ABS_PRESSURE, input_dev->absbit);
input_set_abs_params(input_dev, ABS_MT_POSITION_X, 0, SCREEN_MAX_X, 0, 0);
input_set_abs_params(input_dev,ABS_MT_POSITION_Y, 0, SCREEN_MAX_Y, 0, 0);
input_set_abs_params(input_dev,ABS_MT_TOUCH_MAJOR, 0, PRESS_MAX, 0, 0);
input_set_abs_params(input_dev,ABS_MT_WIDTH_MAJOR, 0, 200, 0, 0);
在work queue里面：
读取的i2c字节比单点触摸的多。
读取的多个字节中，有一个字节，是能反映但是触摸的点数是几点触摸的。
本例驱动支持最多5点触摸
