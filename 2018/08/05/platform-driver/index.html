新的内核有个比较好的功能，就是probe函数，如果以-517也就是-EPROBE_DEFER返回的话，、
-EPROBE_DEFER就是返回值-517，意即/ Driver requests probe retry /
内核会帮助重新安排延后一段时间再probe一次。
因为此时往往是因为probe时间太早，有些相关的资源还没有注册到位导致probe失败。
具体的例子可见：
davinci_evm sound: ASoC: CODEC DAI tlv320aic32x4-hifi not registered
davinci_evm sound: snd_soc_register_card failed (-517)
但后续又初始化了一次，成功了。
davinci_evm sound: tlv320aic32x4-hifi <-> 48038000.mcasp mapping ok
上面的两次信息都是由davinci_evm_probe()打印出来的。
第一次probe是正常执行。
第二次probe的call stack如下，可以看到是内核重新安排的：
