ALSA audio protocol

音频协议很多。包含：
AC97，DSPA,DSPB,I2S,LSB justified,MSB justified,PCM
但这些协议大同小异

目前涉及过的协议是i2s和lsb-j
这两种协议横向比较，基本相同。
相同点：
1.都是下降沿打数据，上升沿采样。
不同点：
1.i2s一上来wclk是用低电平（左声道。此时bclk是下降来来临）表示数据传输的开始。lsb-j一上来wclk是用高电平（左声道.此时bclk是下降来来临）表示数据传输的开始。
2.i2s在bclk数据传输开始的那个下降沿来临后，要多等一个周期，才开始打数据/采数据。lsb-j，在在bclk数据传输开始的那个下降沿来临后，要马上打数据，因为紧接着下一个bclk上升沿，就要采数据了。

至于dsp协议，比较特殊的地方在于，wclk的宽度，不是和整个数据位宽一致，而是一个bit的脉冲。来一次脉冲，意味着接下来要传输完左右声道两组数据。

多通道的wav文件格式：
参考《Multichannel PCM WAV file format for 24 bit 48 kHz 5.1 Multichannel WAV file》
先按照声道由左到右的顺序（slot1~8）一次把当前这次采样点的数据填完，然后再填下一个采样点。

多通道软件拼接方式：
见Asoundrc - AlsaProject.pdf的Virtual multi channel devices章

TDM bclk/采样率/slot数的关系：
256fs其实就是256fs.fs的意思sample rate，256是每次采样，总的bit数：
例如256可以是tdm的32bit 8slot,即256=328
所以256fs=256fs即bclk的频率，即1秒内，总共多少个bit。
例如fs=48khz，此时bit clock应该是48khz 8 slot * 32bit = 12.288mhz
