不同sample rate的音乐，wclk和bclk分频的细节

例子1：
Playing WAVE ‘s.wav’ : Signed 16 bit Little Endian, Rate 44100 Hz, Stereo
sysclk=24000000
bclk_freq=1411200
实测测量到bclk是1.41MHz，wclk是44.1khz
例子2：
Playing WAVE ‘48k.wav’ : Signed 16 bit Little Endian, Rate 48000 Hz, Stereo
sysclk=24000000
bclk_freq=1536000
实测测量到bclk是1.6MHz，wclk是50khz

可以看到，不同的音乐，bclk不一样。wclk也不一样，wclk=sample rate

怎么去理解？
首先。以pt8211这种应用场景为例。
bclk和wclk是由mcasp发出的。
mcasp使用内部24MHz
那么就需要把24MHz进行分频。

bclk_freq也就是bclk的频率，计算的公式是sample_size channels rate
那么，首先，对于wclk，是这样理解的：
平时说的比如192KHz采样频率。是指一秒钟之内，有192k个采样点，而每个采样点，其实包含左声道+右声道两个数据。
所以这个word select input frequency，其实就是Sampling rate frequency。所以测量wclk的频率，就能测量到sample rate。
那么再回到bclk。
bclk就是sample_size（比如是16bit位宽） × channels（比如2声道）× rate（也就是指采样频率）
回到例子1：bclk就是：16244.1Khz=1411.2Khz=1.41MHz

所以，针对24MHz的内部时钟，只要能分频出1.41MHz，就能进一步分频得到wclk（因为bclk再除sample_size 再除 channels就是wclk）
