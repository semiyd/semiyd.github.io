Yocto


演化历史：

buildroot->OpenEmbedded->Yocto

yocto的编译系统是poky.

poky=bitbake+metadata(.conf .bb .bbclass)

以i.mx6平台的yocto编译为例：
apt-get install gawk wget git-core diffstat unzip texinfo gcc-multilib \
build-essential chrpath socat libsdl1.2-dev

apt-get install libsdl1.2-dev xterm sed cvs subversion coreutils texi2html \
docbook-utils python-pysqlite2 help2man make gcc g++ desktop-file-utils \
libgl1-mesa-dev libglu1-mesa-dev mercurial autoconf automake groff curl lzop asciidoc

apt-get install u-boot-tools

mkdir ~/bin
curl http://commondatastorage.googleapis.com/git-repo-downloads/repo > ~/bin/repo
chmod a+x ~/bin/repo

git config –global user.email “dong.yan@xxx.com“
git config –global user.name “Dong Yan”

repo init -u git://git.freescale.com/imx/fsl-arm-yocto-bsp.git -b imx-4.1-krogoth -m imx-4.1.15-2.0.3.xml

repo sync
(first part of repo sync will access google sites.need to do this inside xxx.second part of repo sync need to access git.need to do this at home)

DISTRO=fsl-imx-fb MACHINE=imx6ull14x14evk source fsl-setup-release.sh -b build_result/
umask 022

unpack the /media/hdd/xxx/i.MX6ULL/krogoth/backup/krogoth_downloads.tar
to /media/hdd/xxx/i.MX6ULL/krogoth/fsl-release-bsp and overwrite the existing downloads/ directory.

bitbake core-image-base

the result is in:
/media/hdd/xxx/i.MX6ULL/krogoth/fsl-release-bsp/build_result/tmp/deploy/images/imx6ull14x14evk/
