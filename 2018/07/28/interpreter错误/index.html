interpreter错误

eN-Display执行后，显示-sh: ./eN-Display@: No such file or directory 什么鬼？！
在pc机上，可以使用ldd查看应用依赖的库。
在嵌入式平台上，如果没有现成的ldd的话，其实不用交叉编译，这样就可以：
arm-none-linux-gnueabi-readelf -a xxx_app | grep “Shared library:”
所以结果是：
arm-poky-linux-gnueabi-readelf -a wiggly | grep “Shared library:”
0x00000001 (NEEDED) Shared library: [libts.so.0]
0x00000001 (NEEDED) Shared library: [libQtGui.so.4]
0x00000001 (NEEDED) Shared library: [libQtNetwork.so.4]
0x00000001 (NEEDED) Shared library: [libQtCore.so.4]
0x00000001 (NEEDED) Shared library: [libpthread.so.0]
0x00000001 (NEEDED) Shared library: [libstdc++.so.6]
0x00000001 (NEEDED) Shared library: [libm.so.6]
0x00000001 (NEEDED) Shared library: [libgcc_s.so.1]
0x00000001 (NEEDED) Shared library: [libc.so.6]
但问题是，以上的库，搜了下，其实都是有的。而且其实就算某个库没有，那应该报错缺某个库才对。
并且排除了LD_LIBRARY_PATH的设置问题，设置了也没用。
根据dyj搜到的信息，其实可以进一步看interpreter的依赖情况：
arm-poky-linux-gnueabi-readelf -a wiggly | grep “interpreter”显示：
[Requesting program interpreter: /lib/ld-linux.so.3]
检查了下，其实ld-linux.so.3，和ld-linux-armhf.so.3这类名字一样，一般都需要做软链接指向ld-x.xx.so
如下：
root@myd-y6ull14x14:/home# ls -l /lib/ld*
-rwxr-xr-x 1 root root 134400 May 31 16:03 /lib/ld-2.23.so
lrwxrwxrwx 1 root root 10 Jan 1 1970 /lib/ld-linux-armhf.so.3 -> ld-2.23.so
lrwxrwxrwx 1 root root 10 Jul 5 16:18 /lib/ld-linux.so.3 -> ld-2.23.so
但实际上米尔的rootfs没做这个链接，导致interpreter失败。
同样，eN-Display也是依赖这个ld-linux.so.3，所以也不能跑。
而4.8.7的mainwindows，其interpreter是ld-linux-armhf.so.3，正好米尔的板子上有，所以可以跑。
并且am3354的板子正好只有ld-linux.so.3而没有ld-linux-armhf.so.3，所以跑eN-Display是可以的。

其实之前运行pvrsrvctl的时候，遇到过缺ld-linux-armhf.so.3的情况，也是非常难查。
运行的时候，直接报错：-sh: ./pvrsrvctl: not found
最后strings pvrsrvctl | grep lib才grep到这个库的。
