early_printk

Posted on 2018-07-28 | In Linux , Kernel

early_printk


early_printk()

用于打印kernel启动过程中早期的信息。
内核启动的时候如果停留在底下的log就不动了，该怎么办？没有任何log输出，手足无措。
Starting kernel …
Uncompressing Linux… done, booting the kernel.
这时候early printk就可以派上用场了。
需要重新配置内核：
Kernel hacking —> Kernel debugging
Kernel hacking —> Kernel low-level debugging functions –> Early printk
然后就能early_printk了。

这一功能，需要你的arch实现底层的uart发送函数， 即include/mach/debug-macro.S中的addruart, senduart, waituart, busyuart等函数。

在什么时候可以用printk，或者说在什么时候往前都必须要early_printk?
printk的调用，如果放在kernel启动的早期阶段，最终还是可以打印出来的，只不过会被延后。
所以具体到kernel启动的哪个位置，printk可以正确反映执行的时间点而不会被延后了？
如果什么时候printk和early_printk两者一起被打印了，说明printk没被延后了。
实际实验证明，需要在start_kernel()->console_init()以后的打印信息，可以用printk。
在console_init()之前的信息，都必须用early_printk，否则如果在console_init()之前就死掉的话，就会打印不出来的。

early_printk最早能反映什么时候？
init\Main.c start_kernel()一开始入口的地方

想要比这个还要早就打印信息，怎么办？
不太会遇到，比start_kernel()还要早，就是arch\arm\boot\compressed\Misc.c的decompress_kernel()了。
这里面已不能用early_printk()（编译的时候链接不过），但是可以用putstr()

early_printk和early_print的区别？
kernel代码里面有early_print，从代码上看，仅仅是arch\arm\kernel\Setup.c里面定义的静态函数，别的文件没法调用
具体做的事情相当于early_printk再加一句printk，也就是打印两遍。不去管它，用early_printk就是。
