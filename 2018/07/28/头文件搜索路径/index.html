头文件搜索路径

Posted on 2018-07-28 | In Linux , Kernel

头文件搜索路径

1.用x86的gcc编译application
在虚拟机下面，输入echo ‘main(){}’ | gcc -E -v -
在打印出来的信息中，有这么一段：

    #include "..." search starts here:
    #include <...> search starts here:
     /usr/lib/gcc/i686-linux-gnu/4.8/include
     /usr/local/include
     /usr/lib/gcc/i686-linux-gnu/4.8/include-fixed
     /usr/include/i386-linux-gnu
     /usr/include        ------->主要在这里
    End of search list.
这个就是默认的
一般都是在/usr/include/下面

2.用arm的arm-linux-gcc交叉编译application
在虚拟机下面，输入echo ‘main(){}’ | /usr/local/arm/4.3.2/bin/arm-none-linux-gnueabi-gcc -E -v -
在打印出来的信息中，有这么一段：

    #include "..." search starts here:
    #include <...> search starts here:
     /usr/local/arm/4.3.2/bin/../lib/gcc/arm-none-linux-gnueabi/4.3.2/include
     /usr/local/arm/4.3.2/bin/../lib/gcc/arm-none-linux-gnueabi/4.3.2/include-fixed
     /usr/local/arm/4.3.2/bin/../lib/gcc/arm-none-linux-gnueabi/4.3.2/../../../../arm-none-linux-gnueabi/include
     /usr/local/arm/4.3.2/bin/../arm-none-linux-gnueabi/libc/usr/include        ------->主要在这里
    End of search list.
注意这里的绝对路径 /usr/local/arm/4.3.2/bin/其实是根据，你实际把交叉工具链放在哪里，他就自动从绝对路径（比如 /usr/local/arm/4.3.2/bin/）开始找bin/../arm-none-linux-gnueabi/libc/usr/include
所以总之是在交叉编译工具链里面的（其实主要就是交叉编译工具编的glibc自带的库）

3.编译内核驱动时
默认搜索的路径在内核的include/目录下

见这个例子：
    #include <linux/interrupt.h>
    #include <linux/pm_runtime.h>
    #include <linux/if_vlan.h>
    #include <linux/net_switch_config.h>

    #include <linux/cpsw.h>
    #include <plat/dmtimer.h>
    #include "cpsw_ale.h"
    #include "davinci_cpdma.h"

    头文件分几种。
    1.linux标准的头文件。直接去linux的标准路径下去include。例如#include <linux/interrupt.h>
    2.当前同一目录下的，和具体驱动相关的头文件。使用""来include。例如：#include "cpsw_ale.h"
    3.可能需要被别的目录下的驱动也用到的头文件，则需要放到标准的linux搜索include的路径里面去。例如#include <linux/cpsw.h>，是放在include/linux/下面的。这样
        别的目录下的别的驱动，也可以include到这个cpsw.h

4.通过-I参数特别指定
假设有个头文件，放在/home/new下。想要搜到这个头文件，
那么编译的时候，就要：
arm-linux-gcc -g -o app app.c -I/home/new
此时默认搜索路径依然有效。只是在默认的基础上，再多加个特别指定的路径。
