2.异步IO：
参考 ：https://www.ibm.com/developerworks/linux/library/l-async/
异步IO的基本作用，就是app在请求某种IO操作的时候，无需阻塞或者非阻塞轮询，而是直接走人，自己继续干自己的活，然后异步接收通知，类似上面的异步通知。
只不过上面异步的是通知，这里异步的其实是IO操作。
比较常见的应用场景，是有多个异步的IO需要处理。那么单一的进程，可以在某个IO在内核态自己处理的时候，去关注和处理别的已经完成的IO。

数据结构：
aiocb (AIO I/O Control Block)结构体

那么异步IO分成两大类使用的方法：

一是基于glibc的接口：
参考 Boost application performance using asynchronous I_O.pdf
aio_read Request an asynchronous read operation
aio_error Check the status of an asynchronous request
aio_return Get the return status of a completed asynchronous request
aio_write Request an asynchronous operation
aio_suspend Suspend the calling process until one or more asynchronous requests have completed (or failed)
aio_cancel Cancel an asynchronous I/O request
lio_listio Initiate a list of I/O operations

下面是具体的介绍：
int aio_read( struct aiocb *aiocbp );
此函数调用后立即返回，表示申请已经排入了处理队列中。
