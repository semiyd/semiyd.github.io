内核线程


用于帮助驱动完成一些异步事件的处理(接收异步事件，并调用用户空间的程序)和睡眠等待一些事件的，运行于内核的线程。
常见的已有的内核线程：
[kthreadd]
内核线程的父线程
[ksoftirqd/0]
用于处理中断下半部分，也就是tasklet
[events/0]
用来处理work queue
[pdflush]
批量把页缓冲写入磁盘的后台线程
[kjournald]
日志线程
[nfsd]
Network File System (NFS) 的后台线程
自定义内核线程
通常的使用过程包括：
用kernel_thread建立线程，wake_up_interruptible唤醒这个线程
线程本身如下：(这个例子，线程会等待别人唤醒，应该不属于最简单的内核线程的例子，不知为什么以前记录的这个笔记)
DECLARE_WAITQUEUE
daemonize
allow_signal
add_wait_queue

schedule
至此就是等待到相应的事件了。处理事件。
如果需要，调用用户空间的User Mode Helpers，在用户空间进一步处理事件。
如果收到关线程的信号，跳出循环
set_current_state(TASK_RUNNING);
remove_wait_queue
在上述的基础上，还有一套Kthread的API，进行了进一步的包装，并且集成了completion interface，使用更加简单
通常的使用过程包括：
static struct task_struct MyThread = NULL;
static int MyPrintk(void data)
{
while(!kthread_should_stop())
{
做具体的事情。
}
}
static int init init_kthread(void)
{
MyThread = kthread_run(MyPrintk,”hello world”,”mythread”);
}
static void exit exit_kthread(void)
{
if(MyThread)
{
kthread_stop(MyThread);
}
}
module_init(init_kthread);
module_exit(exit_kthread);
