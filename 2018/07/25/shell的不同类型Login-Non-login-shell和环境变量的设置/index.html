shell的不同类型Login Non-login shell和环境变量的设置

Posted on 2018-07-25 | In Linux , Application , Shell

​ 首先有两个维度：
Login/Non-login shell
Login shell:
就是指板子启动后，输入root按回车登录，然后得到的那种shell。
Non-login shell:
就是指在/etc/init.d/rcS里面自动运行的脚本，这些脚本并未经过登录过程，直接运行。
Interactive/Non-interactive shell
Interactive shell:
是指会接受键盘输入，或者会把打印信息打印到终端的shell命令或者脚本。
Non-interactive shell:
就是指既没有输入的方法，也不会把信息打印到任何终端的shell命令或者脚本。

​ 所以组合如下：
interactive login shell
环境变量取决于/etc/profile或者~/.profile
例如：通过getty,ssh,su等，登录进去，得到的首个shell。
non-interactive login shell
环境变量取决于/etc/profile或者~/.profile
例如：极为罕见，有时用于一些X settings用display manager登录，然后做一些工作，所以无互动，但又是登录了。
interactive, non-login shell
环境变量取决于~/.bashrc或者~/.zshrc等
例如：在已有的session（screen, X terminal, Emacs terminal buffer, a shell inside another, etc.）上开出来的shell
non-interactive, non-login shell
环境变量取决于~/.bashrc或者~/.zshrc等
例如：类似interactive, non-login shell，只是没有互动的输入输出的就算是这类。

​ 所以可以看出，其实是不是interactive无所谓。关键还是，是不是login还是non-login。login就是/etc/profile，non-login就是.bashrc

​ 如何判断当前是不是login shell？
prompt> echo $0
-bash # “-“ is the first character. Therefore, this is a login shell.
prompt> echo $0
bash # “-“ is NOT the first character. This is NOT a login shell.

​ 对于嵌入式板子上的busybox，实测如下:
登录后得到的可以敲命令的shell也就是login-shell，由/etc/profile或者/root/.profile决定环境变量设置。
在已有的shell基础上（例如刚刚登录得到的shell）再运行的shell，也就是non-login shell，由/etc/init.d/rcS决定环境变量设置。
另外，为什么会执行rcS?因为init进程，会去读/etc/inittab里面的sysinit项。找到sysinit对应的值就是/etc/init.d/rcS
