reserved memory

总体介绍：
对内核声明预留内存的区域，内核就不会在malloc的时候触碰。
dts里面配置后，probe的时候用of接口读上来，然后可以用memremap（和ioremap类似）在内核空间用其虚拟地址，也可以在mmap里面映射给用户空间让其使用。
dma_alloc_coherent会优先使用cma区域，cma用完后，会使用非cma的区域分配内存。

一：reserved-memory
dts例子：
reserved-memory {

#address-cells = <2>;

#size-cells = <2>;
ranges;

reserved: buffer@0 {
no-map;
reg = <0x0 0x70000000="" 0x0="" 0x10000000="">;
};
};

reserved-driver@0 {
compatible = “xlnx,reserved-memory”;
memory-region = <&reserved>; //具体的driver声明要使用这块叫reserved的区域
};

驱动例子：
/ Get reserved memory region from Device-tree /
np = of_parse_phandle(dev->of_node, “memory-region”, 0);
if (!np) {
dev_err(dev, “No %s specified\n”, “memory-region”);
goto error1;
}

rc = of_address_to_resource(np, 0, &r);
if (rc) {
dev_err(dev, “No memory address assigned to the region\n”);
goto error1;
}

lp->paddr = r.start;
lp->vaddr = memremap(r.start, resource_size(&r), MEMREMAP_WB);
dev_info(dev, “Allocated reserved memory, vaddr: 0x%0llX, paddr: 0x%0llX\n”, (u64)lp->vaddr, lp->paddr);

注：of_parse_phandle的第三个参数，是查找phandle的索引号。例如6.6.1内核的mt2701.dtsi中，mediatek, larbs = {&larb0 &larb1 &larb2}，需使用最后一个参数决定返回哪个larb。
of_address_to_resource是针对memory-region类型的资源的，即在reserved-memory的dts节点里声明的子节点memory@xxx。其第二个参数index，就是说，如果这个memory-region有多个段，也指明是哪个段。例如memory@xxx内部，有多个reg=<>和reg=<>表示多段没错，则可以用这个index指明。

验证：
cat /proc/iomem能打印出当前reserved memory的区域
root@plnx_aarch64:~# cat /proc/iomem
00000000-6fffffff : System RAM
00080000-00b37fff : Kernel code
011c9000-012b8fff : Kernel data

二：reserved memory用于创建一个dma memory pool，并给dma engine API使用：
dts:
reserved-memory {

#address-cells = <2>;
#size-cells = <2>;
ranges;

reserved: buffer@0 {
   compatible = "shared-dma-pool"; //注意这个shared-dma-pool
   no-map;
   reg = <0x0 0x70000000 0x0 0x10000000>;
};

};

reserved-driver@0 {
compatible = “xlnx,reserved-memory”;
memory-region = <&reserved>;
};
驱动：
/ Initialize reserved memory resources /
rc = of_reserved_mem_device_init(dev);
if(rc) {
dev_err(dev, “Could not get reserved memory\n”);
goto error1;
}

/ Allocate memory /
dma_set_coherent_mask(dev, 0xFFFFFFFF);
lp->vaddr = dma_alloc_coherent(dev, ALLOC_SIZE, &lp->paddr, GFP_KERNEL);
dev_info(dev, “Allocated coherent memory, vaddr: 0x%0llX, paddr: 0x%0llX\n”, (u64)lp->vaddr, lp->paddr);
内核启动打印：
[ 0.000000] Reserved memory: created DMA memory pool at 0x0000000070000000, size 256 MiB
[ 0.000000] Reserved memory: initialized node buffer@0, compatible id shared-dma-pool
[ 0.000000] cma: Reserved 128 MiB at 0x0000000068000000

三：reserved memory用于预留CMA
dts：
reserved-memory {

#address-cells = <2>;
#size-cells = <2>;
ranges;

reserved: buffer@0 {
   compatible = "shared-dma-pool";    
   reusable;
   reg = <0x0 0x70000000 0x0 0x10000000>;
   linux,cma-default; // 注意这个标记cma专用
};

};
内核启动打印：
[ 0.000000] Reserved memory: created CMA memory pool at 0x0000000070000000, size 256 MiB
[ 0.000000] Reserved memory: initialized node buffer@0, compatible id shared-dma-pool

例子代码：
linux-5.9.1\drivers\gpu\drm\msm\adreno\adreno_gpu.c
