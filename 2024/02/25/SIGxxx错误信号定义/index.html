SIGxxx错误信号定义

SIGFPE
算数运算错误。包括浮点错误，被0除错误，等
SIGILL
试图执行的指令是非法的，一般由于可执行文件损坏，或者执行过程中破坏了栈的返回地址，导致后续执行了非法的指令。
SIGSEGV
段错误，操作的内存地址无法对应到实际的物理地址，或者试图去写一个只读的地址。例如操作空指针
SIGBUS
对齐错误，操作的内存地址能对应到实际的物理地址，但是因为不对齐，无法操作。
这个发生的时候，用户态会打印Bus error，原因是访问的内存地址（源/目的地）有不满足对齐要求的情况。例如CMA要求源/目的都是8字节对齐，
如果目的地指针加了偏移，不满足了，memcpy就会Bus error.见下面这个解释：
Bus errors are rare nowadays on x86 and occur when your processor cannot even attempt the memory access requested, typically:
using a processor instruction with an address that does not satisfy its alignment requirements.

Segmentation faults occur when accessing memory which does not belong to your process. They are very common and are typically the result of:
using a pointer to something that was deallocated.
using an uninitialized hence bogus pointer.
using a null pointer.
overflowing a buffer.
https://stackoverflow.com/questions/212466/what-is-a-bus-error-is-it-different-from-a-segmentation-fault
SIGABRT
有应用程序自身触发，调用了abort()
SIGIOT
不常见。类似SIGABRT
SIGTRAP
使用debugger的时候，由设置断点的指令产生。正常运行一般不会遇到。
SIGEMT
Emulator trap; this results from certain unimplemented instructions which might be emulated in software, or the operating system’s failure to properly emulate them.不常见
SIGSYS
系统调用错误
