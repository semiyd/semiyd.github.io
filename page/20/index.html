shell的不同类型Login Non-login shell和环境变量的设置

Posted on 2018-07-25 | In Linux , Application , Shell

​ 首先有两个维度：
Login/Non-login shell
Login shell:
就是指板子启动后，输入root按回车登录，然后得到的那种shell。
Non-login shell:
就是指在/etc/init.d/rcS里面自动运行的脚本，这些脚本并未经过登录过程，直接运行。
Interactive/Non-interactive shell
Interactive shell:
是指会接受键盘输入，或者会把打印信息打印到终端的shell命令或者脚本。
Non-interactive shell:
就是指既没有输入的方法，也不会把信息打印到任何终端的shell命令或者脚本。

main函数入参的处理

Posted on 2018-07-23 | In Linux , Application
main函数入参的处理



如何给一个应用加执行的参数？
比如
./abc –verbose

主要是依靠getopt_long来获取和解析入参。
举例如下：

#include

#include <time.h>

#include <string.h>

#include <stdio.h>

#include “System.h”

#include “version.h”

#include <getopt.h>

static void print_usage(void)
{
(void)fprintf(stderr,
“Usage: NetController \n”);
(void)fprintf(stderr,
“\t\t[ -v ] for verbose output on ring management library\n”);
}

#define SHORTOPTS “hv”

static const struct option longopts[] = {
{ “help”, no_argument, NULL, ‘h’ },
{ “verbose”, no_argument, NULL, ‘v’ },
{ NULL, 0, NULL, 0 }
};

int main(int argc, char* argv) {
time_t rawtime;
struct tm timeinfo;
char buffer[30];
time (&rawtime);
timeinfo = localtime(&rawtime);
strftime(buffer, 30, “%F %T”,timeinfo);
register int op;

while (
    (op = getopt_long(argc, argv, SHORTOPTS, longopts, NULL)) != -1)
    switch (op) {
        case 'v':
            ring_mgmt_debug = true;
            break;

        default:
            print_usage();
            return 0;
    }

//开始正式干活

}
