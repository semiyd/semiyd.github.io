右值引用

参考《从4行代码看右值引用 - 写的很好》

左值是指表达式结束后依然存在的持久化对象，右值是指表达式结束时就不再存在的临时对象。

int&& a = 1; //实质上就是将不具名(匿名)变量1取了个别名.实际上就是给那个临时变量取了个名字。这里a的类型是右值引用类型int &&

左值引用只能绑定左值，右值引用只能绑定右值
int b = 1;
int && c = b; //编译错误！ 不能将一个左值复制给一个右值引用

为啥要搞这种右值引用的概念呢？
其目的在于减少拷贝构造函数的调用次数，优化程序执行效率。
有关拷贝构造函数调用的时机，参考《拷贝构造函数被调用的时机.pdf》
