音频通路配置

有关音频通路的控制。可以参考《Linux\Kernel\驱动\Sound\博客\瑞芯微\ALSA音频通路-瑞芯微.pdf》和《Linux\Kernel\驱动\Sound\博客\sepnic\DAPM之三：audio paths与asound.conf.pdf》

主要可以通过asound.conf来配置通路。
通过配置好asound.conf，上层则可打开asound.conf中定义的虚拟设备，而自动选择相应的音频通道。这是asound.conf很重要的一个作用，

这里会用到一种PCM的plugin,叫hooks plugin。参见《Linux\Kernel\驱动\Sound\博客\AlsaProject\PCM (digital audio) plugins.pdf》
通过hooks，可以打开snd_soc_dapm_route类型的一系列路径上的开关。例如：
pcm.AndroidPlayback_Speaker_normal {
type hooks
slave.pcm {
type hw
card 0
device 0
}
hooks.0 {
type ctl_elems
hook_args [
{
name ‘Left Input PGA Switch’
value true
}
{
name ‘Left Input PGA LINPUT1 Switch’
preserve true
lock true
value true
}
{
name ‘Left Input Mixer Input PGA Switch’
preserve true
lock true
value true
}
{
name ‘Left Output Mixer Left Input Mixer Switch’
preserve true
lock true
value true
}
{
name ‘LINEOUT1 Switch’
value true
}
]
}
}
