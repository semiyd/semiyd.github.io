Posted on 2020-07-12 | In Linux , Kernel , 驱动 , ALSA

kcontrol笔记

一个kcontrol代表着一个mixer（多路输入，一路输出,混合多路输入，到一路输出），或者是一个mux（多路输入，一路输出，只能有一个输入被选中），
又或者是一个音量控制器等等。
其结构体定义是
struct snd_kcontrol_new {
snd_ctl_elem_iface_t iface; / interface identifier /
unsigned int device; / device/client number /
unsigned int subdevice; / subdevice (substream) number /
const unsigned char name; / ASCII name of item /
unsigned int index; / index of item /
unsigned int access; / access rights /
unsigned int count; / count of same elements /
snd_kcontrol_info_t info;
snd_kcontrol_get_t get;
snd_kcontrol_put_t put;
union {
snd_kcontrol_tlv_rw_t c;
const unsigned int p;
} tlv;
unsigned long private_value;
};
所以kcontrol是啥？说白了就是一些对codec的控制单元，一些开关。
通过定义在snd_soc_component_driver.controls中，使用snd_soc_register_component注册到ALSA子系统里面去。

可以看到，非常多。但可以分成这样几类：
SOC_SINGLE
SOC_SINGLE_TLV
SOC_DOUBLE
SOC_DOUBLE_TLV
SOC_ENUM
这都是些常见的帮助定义kcontrol的现成宏定义。一个个来看。

SOC_SINGLE

#define SOC_SINGLE(xname, reg, shift, max, invert)
xname（该控件的名字），reg（该控件对应的寄存器的地址），shift（控制位在寄存器中的位移），max（控件可设置的最大值），invert（设定值是否逻辑取反）
以
SOC_SINGLE(“Mic Boost (+20dB) Switch”, AC97_LINE, 5, 1, 0),
为例。
控件的名字是Mic Boost (+20dB) Switch。顾名思义是麦克风增益提升20分贝的一个开关。
寄存器地址是AC97_LINE，定义于include\sound\ac97\regs.h

#define AC97_LINE 0x10 / Line In Volume /
原因是wm9713是符合AC97标准的一个声卡。所谓符合标准，就是其寄存器的布局，是按照AC97标准布局的。
见wm9713.pdf的page88 register map
0x10对应的是MIC Routing寄存器
控制位在寄存器中的位移是bit5，对应手册page90的MIC2MBST这个LABEL。即Enables 20dB gain boost at mono mixer for MICA and MICB
控件可设置的最大值是1，未取反。
所以说白了，这东西就是一开关。
另外，很多个SOC_SINGLE组合起来，也可以用来代表一个mixer单元，也就是多路输入，一路输出的选择器。注意mixer的多路输入，是可以多路同时开启的。
最终mix（混音）后，再通过一路输出出来。
我们会为mixer的每个输入端都单独定义一个简单控件来控制该路输入的开启和关闭，反应在代码上，就是定义一个soc_kcontrol_new数组：
static const struct snd_kcontrol_new left_speaker_mixer[] = {
SOC_SINGLE(“Input Switch”, WM8993_SPEAKER_MIXER, 7, 1, 0),
SOC_SINGLE(“IN1LP Switch”, WM8993_SPEAKER_MIXER, 5, 1, 0),
SOC_SINGLE(“Output Switch”, WM8993_SPEAKER_MIXER, 3, 1, 0),
SOC_SINGLE(“DAC Switch”, WM8993_SPEAKER_MIXER, 6, 1, 0),
};

SOC_DOUBLE
和SOC_SINGLE一样，区别是SOC_SINGLE只控制一个变量，而SOC_DOUBLE则可以同时在一个寄存器中控制两个相似的变量，最常用的就是用于一些立体声的控件。

#define SOC_DOUBLE(xname, reg, shift_left, shift_right, max, invert)
可以看到，SOC_SINGLE的shift演变成了shift_left, shift_right,
以
SOC_DOUBLE(“Headphone Playback ZC Switch”, AC97_HEADPHONE, 14, 6, 1, 0),
为例
控件的名字是Headphone Playback ZC Switch。
寄存器地址是AC97_HEADPHONE也就是0x4。
见wm9713.pdf的page88 register map
0x04对应的是Headphone Volume寄存器
控制位在寄存器中的位移是bit14和bit6，对应手册page51的ZCL和ZCR这两个LABEL。
控件可设置的最大值是1，未取反。
在用户空间，通过amixer controls命令，就可以看到当前已经注册的kcontrol：

amixer controls

numid=30,iface=MIXER,name=’Headphone Playback ZC Switch’

SOC_SINGLE_TLV

#define SOC_SINGLE_TLV(xname, reg, shift, max, invert, tlv_array)
SOC_SINGLE_TLV是SOC_SINGLE的一种扩展，主要用于定义那些有增益控制的控件，例如音量控制器，EQ均衡器等等。
入参和SOC_SINGLE一样，唯一不同的是增加了一个tlv_array参数。
所谓tlv，就是Type-Lenght-Value的意思，数组的第0各元素代表数据的类型，第1个元素代表数据的长度，第三个元素和之后的元素保存该变量的数据。
以
SOC_SINGLE_TLV(“Mic 1 Volume”, AC97_MIC, 8, 31, 1, main_tlv),
为例。
AC97_MIC地址是0x0E。见wm9713.pdf的page45 MIC PGA Volume
shift=8，也就是12:8bit这段，MICAVOL也就是MICA input gain。
最大值是31，也就是11111‘b。
而main_tlv的定义在：
static const DECLARE_TLV_DB_SCALE(main_tlv, -3450, 150, 0);
DECLARE_TLV_DB_SCALE的定义是：

#define DECLARE_TLV_DB_SCALE SNDRV_CTL_TLVD_DECLARE_DB_SCALE

#define SNDRV_CTL_TLVD_DECLARE_DB_SCALE(name, min, step, mute) ，内部tlv的类型是SNDRV_CTL_TLVT_DB_SCALE。
顾名思义，这个是用于dB值定义的tlv，具体内部定义比较复杂，现在只需要知道：
name名字就是main_tlv
min就是寄存器的最小值对应是-3450（实际上对应寄存器是-34.5dB）
step也就是步进的大小，寄存器每增加一个单位值，对应的dB数增加是：
150*0.01dB=1.5dB
所以说白了，这种db scale的tlv，就是用来描述音量旋钮的。最小值多少，步进是多少。

SOC_DOUBLE_TLV
这个就是SOC_SINGLE_TLV的双声道版本。

SOC_ENUM
和mixer类型对应，这个属于mux类型，也就是多路输入，一路输出。但输出只能有一个多路输入被选中。

#define SOC_ENUM(xname, xenum)
以
SOC_ENUM(“ALC Function”, wm9713_enum[6]),
为例
wm9713_enum的定义是：

可以看到，xenum是个soc_enum{}的大集合。SOC_ENUM需要指定是xenum里面的哪个元素。
那第六个元素是：
SOC_ENUM_SINGLE(AC97_PCI_SVID, 14, 4, wm9713_alc_select), / alc source select 6/

#define SOC_ENUM_SINGLE(xreg, xshift, xmax, xtexts)
xreg是AC97_PCI_SVID是0x62，对应wm9713.pdf的page100的0x62寄存器的ALCSEL部分。
这个寄存器的解释是：Controls which channel ALC operates on. 00=none, 01=right only, 10=left only, 11=both
xshift是14，因为寄存器的位的范围是15:14
xmax是4。（实际值最大也就是3）
xtexts是wm9713_alc_select，也就是：
static const char *wm9713_alc_select[] = {“None”, “Left”, “Right”, “Stereo”};
所以就很清楚了。这就是个选择器。从”None”, “Left”, “Right”, “Stereo”中选一个。正好对应0,1,2,3的寄存器值。

除了以上wm9713.c用到的kcontrol，还有这些：

SOC_SINGLE_EXT
这个和SOC_SINGLE类似，但不同的在于可以自己定义get和put回调函数。
get和set是啥？
回到原始的SOC_SINGLE，具体定义如下：

#define SOC_SINGLE(xname, reg, shift, max, invert) \
{ .iface = SNDRV_CTL_ELEM_IFACE_MIXER, .name = xname, \
.info = snd_soc_info_volsw, .get = snd_soc_get_volsw,\
.put = snd_soc_put_volsw, \
.private_value = SOC_SINGLE_VALUE(reg, shift, max, invert) }
可以看到，有两个默认的get和put函数。
我们打开看下get和put函数：
int snd_soc_get_volsw(struct snd_kcontrol kcontrol,
struct snd_ctl_elem_value ucontrol)
{
…
ret = snd_soc_read_signed(component, reg, mask, shift, sign_bit, &val);
…
}
int snd_soc_put_volsw(struct snd_kcontrol kcontrol,
struct snd_ctl_elem_value ucontrol)
{
…
err = snd_soc_component_update_bits(component, reg, val_mask, val);
…
}
此处省去很多细节，只关注一点：get/put函数，其本质是通过snd_soc_read_signed和snd_soc_component_update_bits这两个一读一写的函数，去更新寄存器的值的。
以wm5102.c的代码为例：
SOC_SINGLE_EXT(“Output Compensation Switch”, 0, 0, 1, 0,
wm5102_out_comp_switch_get, wm5102_out_comp_switch_put),
打开wm5102_out_comp_switch_get：
static int wm5102_out_comp_switch_get(struct snd_kcontrol kcontrol,
struct snd_ctl_elem_value ucontrol)
{
…
ucontrol->value.integer.value[0] = arizona->dac_comp_enabled;
…
}
可以看到，根本没访问寄存器。也正是这种非常规的情况，适合自己定义get和put函数。

SOC_DOUBLE_EXT
这个就是SOC_SINGLE_EXT的双声道版本。

所以EXT结尾是一大类，例如需要自己定义get和put回调时，可以使用以下这些带EXT的版本：
SOC_SINGLE_EXT
SOC_DOUBLE_EXT
SOC_SINGLE_EXT_TLV
SOC_DOUBLE_EXT_TLV
SOC_DOUBLE_R_EXT_TLV
SOC_ENUM_EXT
SND_SOC_BYTES_EXT

SND_SOC_BYTES
定义：#define SND_SOC_BYTES(xname, xbase, xregs)
以wm8998.c的
SND_SOC_BYTES(“EQ1 Coefficients”, ARIZONA_EQ1_3, 19),
为例：

#define ARIZONA_EQ1_3 0xE12
参考WM8998.pdf page65
R3602这个寄存器，地址就是0E12h
其含义是EQ1 Frequency Coefficients
并且这个寄存器的范围不止一个，而是R3602(0E12h)一直到R3620(E24h)，一共19个。
所以SND_SOC_BYTES的含义就是，一系列同一功能，但连续很多字节的寄存器。

|->snd_ctl_find_id-遍历kcontrol链表找到与给定id相匹配的kctl
