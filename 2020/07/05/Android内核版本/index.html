Android内核版本

此处首先主要介绍android中有关内核的背景知识。

首先是参考《kernel版本介绍.pdf》，介绍了内核版本和更新的情况。
简单的说，目前分两大块。
一个是stable release，对应大版本的升级，例如3.2，3.8，4.4，4.14等
另外一个是LTS，也就是特定的内核版本，linux社区会规定在特定的三五年内，仍然保持维护，定期添加内核的补丁。（《Active kernel releases》里面的Longterm一节所述）
例如4.4.x，后面的小版本，就是LTS的版本号。

对于SOC厂家来说，升级内核大版本，风险比较大。
所以选择某个LTS，然后使用其定期的补丁，升级小版本，对内核稳定性和测试方面都比较有利。

然后就涉及到了Android Common Kernels，也就是android定制的内核，他是基于LTS来修改出来的。
其源码在：
https://android.googlesource.com/kernel/common/ （需翻墙）
android定制的内核，会根据android需要，增加一些新的模块。(内核中会有CONFIG_ANDROID=y)
并且删除一些针对android设备无用的功能。
和LTS本体相比，android对其的改动大致是：
19.8% Energy Aware Scheduling (kernel/sched)
13.8% Networking (net/netfilter)
13.5% Sdcardfs (fs/sdcardfs)
9.4% USB (drivers/usb)
7.2% SoC (arch/arm64, arch/x86)
6.2% f2fs (fs/f2fs – backports from upstream)
6.1% Input (drivers/input/misc)
5.4% FIQ Debugger (drivers/staging/android/fiq_debugger)
3.6% Goldfish Emulator (drivers/platform/goldfish)
3.4% Verity (drivers/md)
11.6% Other
因为基于LTS，当然定期也会由LTS的补丁可以打上去，并在这个基础上需要通过VTS/CTS（https://qa-reports.linaro.org/lkft/）和build/boot测试验证（kernelci，https://kernelci.org/job/android/）。
从最原始的LTS，到最后OEM/ODM厂家使用的内核，其演变顺序是：
LTS kernel 4.4.y
ASOP common kernel android-4.4
ASOP N 7.0的内核android-4.4-n
ASOP N 7.0的release版内核android-4.4-n-release
android-4.4-n-mr1
android-4.4-n-mr2
