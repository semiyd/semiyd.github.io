perf 笔记

perf编译
menuconfig选上：
General setup —>Kernel Performance Events And Counters —>[*] Kernel performance events and counters
CONFIG_FRAME_POINTER=y
CONFIG_HAVE_PERF_EVENTS=y
 CONFIG_PERF_USE_VMALLOC=y
 CONFIG_PERF_EVENTS=y
 CONFIG_DEBUG_PERF_USE_VMALLOC=y
 CONFIG_VM_EVENT_COUNTERS=y
 CONFIG_PROFILING=y
CONFIG_HW_PERF_EVENTS=y(需要CONFIG_CPU_HAS_PMU支持)
然后下载到板子
编译zlib
编译elfutils
下载elfutils-0.159 elfutils
./configure –host=arm-linux
make
make install DESTDIR=$PWD/../elfutils-build
编译binutils
下载binutils-2.28
mkdir binutils-compile
cd binutils-compile
../binutils-2.28/configure –target=arm-linux –host=arm-linux
make
make install DESTDIR=$PWD/../binutils-build
cp binutils-compile/libiberty/libiberty.a binutils-build/usr/local/lib/
编译perf
修改Makefile：
搜# Define WERROR=0 to disable treating any warnings as errors.
加入WERROR=0
然后改动已有的行：
EXTLIBS = -lpthread -lrt -lelf -lm -lebl -ldl
ALL_CFLAGS = $(CFLAGS) -D_LARGEFILE64_SOURCE -D_FILE_OFFSET_BITS=64 -I/home/yandong/NDC/06.00.00.00/arm-none-linux-gnueabi-4.9.1/tmp/perf/elfutils-build/usr/local/include -I/home/mgc/embedded/codebench/arm-none-linux-gnueabi/libc/usr/include/ -L/home/yandong/NDC/06.00.00.00/arm-none-linux-gnueabi-4.9.1/tmp/perf/elfutils-build/usr/local/lib -L/home/mgc/embedded/codebench/arm-none-linux-gnueabi/libc/armv7-a-neon/lib/ -L/home/yandong/NDC/06.00.00.00/arm-none-linux-gnueabi-4.9.1/tmp/perf/binutils-build/usr/local/lib/

———————————————————————–perf record———————————————————————–
生成报告，并可以解析报告文件
假设现在有个测试程序loop2的代码如下：
//test.c
void longa()
{
int i,j;
for(i = 0; i < 1000000; i++)
j=i; //am I silly or crazy? I feel boring and desperate.
}
void foo2()
{
int i;
for(i=0 ; i < 10; i++)
longa();
}
void foo1()
{
int i;
for(i = 0; i< 100; i++)
longa();
}
int main(void)
{
foo1();
foo2();
}
注意，loop2的symbol不能去掉，编译的时候不能strip。
[root@NDC ~]# perf record -e cpu-clock ./loop2
[ perf record: Woken up 1 times to write data ]

———————————————————————–perf probe———————————————————————–
用于动态的在任意内核代码，加入监视点。例如：
利用 probe 命令在内核函数 schedule() 的第 12 行处加入了一个动态 probe 点，
和 tracepoint 的功能一样，内核一旦运行到该 probe 点时，便会通知 perf。可以理
解为动态增加了一个新的 tracepoint。

[root@ovispoly perftest]# perf probe schedule:12 cpu
Added new event:
probe:schedule (on schedule+52 with cpu)
You can now use it on all perf tools, such as:
perf record -e probe:schedule -a sleep 1
[root@ovispoly perftest]# perf record -e probe:schedule -a sleep 1
Error, output file perf.data exists, use -A to append or -f to
overwrite.
[root@ovispoly perftest]# perf record -f -e probe:schedule -a sleep 1
[ perf record: Woken up 1 times to write data ]

———————————————————————–Perf sched———————————————————————–
这个需要PMU支持，如果perf list支持sched测试项的话，可以测试各个任务之间调度的详情。可惜我的板子无法看到。

perf sched record # low-overhead recording of arbitrary workloads

perf sched map # show summary/map of context-switching
perf sched trace # output finegrained trace
perf sched replay # replay a captured workload using
