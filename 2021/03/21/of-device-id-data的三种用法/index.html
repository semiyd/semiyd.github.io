of_device_id.data的三种用法

of_device_id.compatible这个最常用，就是dts里面用来匹配的字符串。
而of_device_id.data则属于一个可以自定义数据类型的私有变量。

/*

  • Struct used for matching a device
    /
    struct of_device_id {
    char name[32];
    char type[32];
    char compatible[128];
    const void     data;
    };

作用有三：
1.data被赋值成硬件版本号。同一套驱动代码，兼容不同版本的硬件。这样初始化的时候可以区分不同的流程。
2.data被赋值成回调函数。不同的dts字串，对应不同的回调函数具体实现，但回调函数名不变。这样驱动代码
在兼容不同设备的时候，使代码流程简洁，不用判断当前代码是跑在什么版本的设备上。
3.data被赋值成回调函数+硬件配置值的集合。这个和刚刚差不多，只是除了回调，再多加一些私有的变量，同样
在兼容不同设备的时候，使代码流程简洁，不用判断当前代码是跑在什么版本的设备上。

例子详见《Support of different hardware versions in a single driver》
